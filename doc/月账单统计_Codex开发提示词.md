# 月账单统计功能——Codex 开发提示词

你现在需要在现有的家庭记账微信小程序项目中，实现“月账单统计”功能。

请先完整阅读项目代码和同目录下的《月账单统计详细需求文档.md》，再开始修改。

不要直接按照假设创建新项目，也不要跳过现有代码分析。

---

## 一、开发目标

实现一个参考微信账单信息结构、但以现有账单类别为核心的月账单统计功能。

本期只实现月账单，不实现年度账单。

需要支持：

- 月份选择
- 支出和收入切换
- 月度金额总览
- 月度账单笔数
- 最近六个月金额趋势
- 分类金额与占比
- 分类账单明细
- 本月最近 5 条账单
- 本月全部账单
- 新增、编辑、删除账单后的统计刷新
- 加载、空数据和异常状态

## 四、功能要求

### 1. 月份选择

页面默认展示当前月份。

用户可以选择其他年月，但不能选择未来月份。

月份切换后刷新：

- 汇总数据
- 最近六个月趋势
- 分类统计
- 最近账单
- 全部账单查询条件

月份切换时保持当前收支类型。

### 2. 支出和收入切换

提供：

- 支出
- 收入

默认选中支出。

选中支出后展示支出汇总、趋势、分类和账单。

选中收入后展示收入汇总、趋势、分类和账单。

### 3. 月度汇总

展示：

- 当前类型总金额
- 当前类型账单笔数
- 本月支出总额
- 本月收入总额

金额必须使用项目现有格式化规则。

页面不得出现：

- undefined
- null
- NaN

### 4. 最近六个月趋势

以当前选择月份为结束月份，向前计算六个自然月。

必须支持跨年。

示例：

```text
当前月份：2026年2月
趋势月份：2025年9月到2026年2月
```

使用简单柱状图。

优先复用现有图表库。

如果项目没有图表库，优先使用轻量实现，不要为了一个简单柱状图引入大型图表依赖。

点击某个月份柱体后：

- 切换到该月份
- 刷新月账单
- 保持当前收支类型

金额为 0 的月份也需要显示。

### 5. 分类统计

复用现有分类体系。

每个类别展示：

- 图标
- 名称
- 金额
- 占比
- 比例条
- 点击入口

按照金额从高到低排序。

占比计算：

```text
类别金额 / 当前类型总金额 * 100
```

需要处理总金额为 0 的情况。

无分类账单、引用已删除分类的账单，第一版本归入“其他”。

不得丢失这些账单的统计金额。

### 6. 分类账单明细

点击分类后进入分类账单明细页。

查询条件包括：

- 年
- 月
- 收支类型
- 分类 ID

页面顶部展示：

- 年月
- 分类名称
- 分类图标
- 分类总金额
- 分类账单笔数
- 分类占比

下方展示该分类的全部账单。

账单按照实际发生时间倒序。

点击账单复用现有账单详情或编辑页面。

### 7. 最近账单

月账单页面默认展示当前月份、当前类型最近 5 条账单。

每条显示：

- 分类图标
- 账单名称
- 分类名称
- 发生时间
- 备注摘要
- 金额

支出显示负号，收入显示正号。

底部提供“查看全部账单”。

### 8. 全部账单

查看全部后进入本月全部账单页面。

只展示当前：

- 年
- 月
- 收支类型

优先复用现有账单列表。

建议按照日期分组。

分页或上拉加载方式遵循现有项目。

### 9. 数据刷新

以下操作后返回统计页必须刷新：

- 新增账单
- 编辑账单
- 删除账单
- 修改金额
- 修改收支类型
- 修改分类
- 修改发生时间

优先保证数据准确，不要过早设计复杂缓存。

## 五、时间与金额规则

### 1. 时间字段

统计必须使用账单实际发生时间。

不要使用创建时间或更新时间代替，除非现有旧数据确实没有发生时间字段。

### 2. 月份范围

查询月份时使用：

```text
occurredAt >= 当前月第一天 00:00:00
occurredAt < 下个月第一天 00:00:00
```

不要使用月底 23:59:59。

### 3. 时区

沿用项目现有业务时区。

必须检查数据库时间、接口时间和小程序本地时间之间的转换，避免月初月末账单跨月。

### 4. 金额精度

检查现有项目金额存储方式。

如果金额以“分”存储，统计过程中保持整数计算。

如果使用 Decimal 或高精度库，继续复用。

不要直接使用浮点数反复累加产生精度问题。

## 六、推荐接口设计

优先检查现有后端能力，不要先入为主新增接口。

如果需要新增，可以实现类似：

```http
GET /statistics/monthly
```

参数：

```text
year
month
type
```

建议一次返回：

- 当前月份汇总
- 最近六个月趋势
- 分类统计
- 最近 5 条账单

参考返回结构：

```ts
interface MonthlyBillStatistics {
  year: number
  month: number
  type: 'expense' | 'income'

  summary: {
    currentTypeAmount: number
    currentTypeCount: number
    expenseAmount: number
    expenseCount: number
    incomeAmount: number
    incomeCount: number
  }

  trend: Array<{
    year: number
    month: number
    amount: number
    count: number
  }>

  categories: Array<{
    categoryId: string
    categoryName: string
    categoryIcon?: string
    categoryColor?: string
    amount: number
    count: number
    percentage: number
  }>

  recentBills: Array<{
    id: string
    type: 'expense' | 'income'
    amount: number
    categoryId?: string
    categoryName: string
    categoryIcon?: string
    title: string
    remark?: string
    occurredAt: string
  }>
}
```

这只是参考结构。

必须根据现有项目字段进行适配。

分类明细和全部账单优先复用现有账单查询接口。

## 七、页面状态

### 1. 首次加载

展示项目现有骨架屏或 Loading。

至少覆盖：

- 汇总
- 趋势图
- 分类统计
- 最近账单

### 2. 空数据

本月无账单时显示：

```text
本月还没有账单
记一笔后，这里会生成月度统计
```

可以提供“去记一笔”按钮，并跳转到现有记账入口。

当前类型无数据时，例如没有收入，显示：

```text
本月还没有收入记录
```

趋势图仍保留六个月份。

### 3. 请求失败

显示：

```text
月账单加载失败，请稍后重试
```

提供“重新加载”。

不要无限自动重试。

## 八、请求竞态处理

用户可能快速切换月份和收支类型。

必须避免：

- 较早请求覆盖较晚请求
- 页面销毁后继续更新状态
- 同一个条件重复发起大量请求
- 快速点击导致页面数据显示错乱

根据现有技术栈选择合适方式：

- 请求 ID
- AbortController
- 取消请求
- latest-only 标记
- 页面生命周期判断

不要引入与项目不匹配的复杂方案。

整体参考微信账单的信息层级，但不要复制微信样式。

要求：

- 保持现有小程序主题
- 金额信息最突出
- 图表简单
- 分类信息对齐
- 不使用复杂动画
- 不使用 3D 图表

- 适配小屏和安全区域
- 类别名称过长时合理截断
- 大金额不得溢出
