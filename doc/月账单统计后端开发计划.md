# 月账单统计后端开发计划

## 1. 开发原则

- 只开发后端统计能力，不改动无关业务。
- 优先复用现有 `Bill`、`Category` 模型。
- 不新增重复账单模型或分类体系。
- 金额聚合沿用现有“分”的业务口径，使用整数分计算，避免浮点误差。
- 保持现有接口风格、鉴权方式和响应格式。

## 2. 实施步骤

### 步骤一：项目分析

已确认：

- 后端为 NestJS + TypeORM + MySQL。
- 全局前缀为 `/api`。
- 账单实体为 `Bill`，金额字段为 `DECIMAL(10,2)`，当前业务单位为分。
- 账单实际发生日期字段为 `billDate`，当前只有日期，没有时分秒。
- 收支类型枚举为 `income` / `expense`。
- 删除账单通过 `isDeleted` 软删除。
- 分类实体为 `Category`，包含名称、图标、类型、排序和系统分类标记。
- 分类当前没有颜色字段和软删除字段。
- 现有 `/api/bills/statistic` 是年度简版统计，不满足月账单需求。

### 步骤二：新增统计模块

新增文件：

- `src/statistics/statistics.module.ts`
- `src/statistics/statistics.controller.ts`
- `src/statistics/statistics.service.ts`
- `src/statistics/dto/query-monthly-statistics.dto.ts`

修改文件：

- `src/app.module.ts`
- `src/bill/dto/query-bill.dto.ts`
- `src/bill/bill.service.ts`

### 步骤三：实现查询参数校验

实现 `QueryMonthlyStatisticsDto`：

- `year`：整数，范围 1970 到 9999
- `month`：整数，范围 1 到 12
- `type`：`BillType`

服务层额外校验不允许查询未来月份。

### 步骤四：实现月度汇总

查询当前月份全部有效账单，并计算：

- 支出总额、支出笔数
- 收入总额、收入笔数
- 当前类型总额、当前类型笔数

金额按整数分计算，并继续按分返回，前端展示时再转换为元。

### 步骤五：实现最近六个月趋势

计算结束月份向前 6 个自然月。

查询这 6 个月内当前类型的有效账单，按月份聚合金额和笔数。

没有账单的月份返回 0。

### 步骤六：实现分类统计

基于当前月份、当前类型账单聚合分类数据。

处理逻辑：

- 关联到有效分类时使用分类名称、图标、排序。
- 关联不到分类时归入“其他”。
- 按金额、笔数、分类排序、分类 ID 稳定排序。
- 占比按 `分类金额 / 当前类型总金额 * 100` 计算。

### 步骤七：实现最近账单

查询当前月份、当前类型最近 5 条有效账单。

排序：

- `billDate DESC`
- `createdAt DESC`

返回分类名称、图标、备注、金额、日期等前端展示所需字段。

### 步骤八：注册接口

新增接口：

```http
GET /api/statistics/monthly?year=2026&month=6&type=expense
```

接口需要 JWT 鉴权。

### 步骤九：扩展账单列表筛选

复用现有接口：

```http
GET /api/bills
```

新增支持：

- `type`
- `categoryId`

这样本月全部账单页和分类账单明细页可以直接复用账单列表接口。

### 步骤十：验证

运行：

```bash
npm run build
npm run lint
```

如项目现有测试可运行，再执行：

```bash
npm test
```

## 3. 验收清单

- 查询当前月支出统计成功。
- 查询当前月收入统计成功。
- 查询未来月份返回参数错误。
- 最近六个月支持跨年。
- 没有账单的趋势月份返回 0。
- 当前类型没有账单时金额和笔数为 0。
- 分类金额合计等于当前类型总金额。
- 无法关联分类的账单归入“其他”。
- 最近账单最多返回 5 条。
- 软删除账单不进入统计。
- `/api/bills` 支持按月份、类型和分类组合筛选。
- 构建通过。
- Lint 通过或仅保留项目既有问题。
