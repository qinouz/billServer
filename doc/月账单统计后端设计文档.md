# 月账单统计后端设计文档

## 1. 目标

在现有 NestJS 后端中提供月账单统计能力，支撑小程序统计页展示指定月份的收支总览、最近六个月趋势、分类统计和最近账单。

本期只实现月账单统计，不实现年度账单、同比环比、预算、AI 分析、导出分享等功能。

## 2. 当前后端现状

### 2.1 技术栈

- 框架：NestJS 11
- ORM：TypeORM 0.3
- 数据库：MySQL
- 鉴权：JWT，接口通过 `JwtAuthGuard` 获取当前用户
- 响应格式：全局 `TransformInterceptor` 包装为 `{ code, data, message, path }`
- 参数校验：全局 `ValidationPipe`，开启 `whitelist`、`transform`、`forbidNonWhitelisted`

### 2.2 账单模型

账单实体为 `Bill`，表名 `bills`。

关键字段：

- `userId`：用户 ID
- `categoryId`：分类 ID
- `amount`：数据库实体字段名，数据库类型 `DECIMAL(10,2)`，仅作为兼容既有表结构使用；对外接口统一使用 `amountCents`
- `type`：收支类型，枚举值为 `income` / `expense`
- `billDate`：账单实际发生日期，数据库类型 `DATE`
- `isDeleted`：软删除标记
- `createdAt` / `updatedAt`：创建和更新时间

当前模型只有日期，没有时分秒。因此本期统计和排序以 `billDate` 作为实际发生时间字段；同一天内再按 `createdAt` 倒序。

### 2.3 分类模型

分类实体为 `Category`，表名 `categories`。

关键字段：

- `id`
- `userId`：为空表示系统分类
- `name`
- `icon`
- `type`
- `sortOrder`
- `isSystem`

当前分类模型没有颜色字段，也没有软删除字段。自定义分类如果已被未删除账单使用，当前后端禁止删除；系统分类清理时会保留仍被账单引用的旧分类。

### 2.4 现有接口能力

现有账单列表接口：

```http
GET /api/bills?month=2026-06&type=expense&categoryId=1&pageNo=1&pageSize=20
```

本期扩展后支持按月份、收支类型、分类和分页查询，可复用在分类账单明细页和本月全部账单页。

## 3. 新增接口

### 3.1 月账单统计

```http
GET /api/statistics/monthly
```

鉴权：需要 JWT。

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| year | number | 是 | 年份，例如 `2026` |
| month | number | 是 | 月份，1 到 12 |
| type | string | 是 | `expense` 或 `income` |

限制：

- 不允许查询未来月份。
- 年份范围限制为 1970 到 9999。
- `type` 必须是现有 `BillType` 枚举值。

### 3.2 返回结构

接口返回业务数据如下，外层仍由全局拦截器包装。

```ts
interface MonthlyStatisticsResponse {
  year: number
  month: number
  type: 'expense' | 'income'
  summary: {
    /** 当前收支类型总金额，单位：分 */
    currentTypeAmountCents: number
    currentTypeCount: number
    /** 本月支出总金额，单位：分 */
    expenseAmountCents: number
    expenseCount: number
    /** 本月收入总金额，单位：分 */
    incomeAmountCents: number
    incomeCount: number
  }
  trend: Array<{
    year: number
    month: number
    /** 当月当前收支类型总金额，单位：分 */
    amountCents: number
    count: number
  }>
  categories: Array<{
    categoryId: string
    categoryName: string
    categoryIcon: string
    categoryColor?: string
    /** 当前分类总金额，单位：分 */
    amountCents: number
    count: number
    /** 当前分类占当前类型总金额的百分比，例如 39.35 表示 39.35% */
    percentage: number
  }>
  recentBills: Array<{
    id: string
    type: 'expense' | 'income'
    /** 单笔账单金额，单位：分 */
    amountCents: number
    categoryId: string
    categoryName: string
    categoryIcon: string
    title: string
    remark: string
    occurredAt: string
    billDate: string
    createdAt: Date
  }>
}
```

说明：

- 对外金额字段统一命名为 `amountCents` 或 `xxxAmountCents`，均返回正数，单位为“分”；前端展示为人民币时再转换为“元”，并根据 `type` 决定正负号。
- `categoryColor` 当前后端无来源，暂不返回或返回 `undefined`。
- `title` 当前后端无独立账单名称字段，先使用分类名称作为标题。
- `occurredAt` 当前使用 `billDate`，格式为 `YYYY-MM-DD`。

### 3.3 账单列表筛选扩展

复用现有账单列表接口：

```http
GET /api/bills
```

新增可选查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| month | string | 否 | 月份，格式 `YYYY-MM` |
| type | string | 否 | `expense` 或 `income` |
| categoryId | string | 否 | 分类 ID |
| pageNo | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页数量，默认 20，最大 100 |

用于本月全部账单：

```http
GET /api/bills?month=2026-06&type=expense&pageNo=1&pageSize=20
```

用于分类账单明细：

```http
GET /api/bills?month=2026-06&type=expense&categoryId=1&pageNo=1&pageSize=20
```

## 4. 数据口径

### 4.1 有效账单

只统计满足以下条件的账单：

- `userId` 等于当前登录用户
- `isDeleted = false`

### 4.2 月份范围

由于当前 `billDate` 是 DATE 字段，统计月份使用日期闭区间：

```text
billDate >= YYYY-MM-01
billDate <= YYYY-MM-lastDay
```

该实现与需求中的半开区间意图一致：覆盖自然月内所有日期。后续如果账单发生时间升级为 DATETIME，应改为：

```text
occurredAt >= 当前月第一天 00:00:00
occurredAt < 下个月第一天 00:00:00
```

### 4.3 金额精度

数据库金额字段仍为 `DECIMAL(10,2)`，TypeORM 读取为字符串；但当前业务已经按“分”传递和返回金额。

后端聚合时统一转换为整数分进行加总：

```text
"128050.00" -> 128050
```

最终仍按“分”返回给前端，避免破坏现有接口金额口径。前端展示金额时负责除以 100 并格式化为元。

### 4.4 最近六个月趋势

以查询的 `year`、`month` 为结束月份，向前包含六个自然月。

示例：

- 查询 2026 年 6 月：返回 2026 年 1 月到 2026 年 6 月
- 查询 2026 年 2 月：返回 2025 年 9 月到 2026 年 2 月

没有账单的月份也返回，金额和笔数为 0。

### 4.5 分类统计

分类统计只统计当前查询类型：

- `type=expense` 时只统计支出分类
- `type=income` 时只统计收入分类

排序规则：

1. 金额从高到低
2. 笔数从高到低
3. 分类 `sortOrder` 从小到大
4. 分类 ID 从小到大

当前账单 `categoryId` 非空且数据库层没有外键约束。若发生分类缺失或无法关联分类，归入“其他”，避免账单金额丢失。

### 4.6 最近账单

返回当前月份、当前收支类型最近 5 条有效账单。

排序：

1. `billDate` 倒序
2. `createdAt` 倒序

## 5. 模块设计

新增后端模块：

```text
src/statistics/
  statistics.module.ts
  statistics.controller.ts
  statistics.service.ts
  dto/query-monthly-statistics.dto.ts
```

模块依赖：

- `Bill` repository
- `Category` repository

`AppModule` 注册 `StatisticsModule`。

## 6. 错误处理

参数错误使用 `BadRequestException`。

典型错误：

- `year` 缺失或非法
- `month` 不在 1 到 12
- `type` 不是 `income` / `expense`
- 查询未来月份

接口异常由全局异常过滤器统一输出。

## 7. 后续扩展

本期新增月账单统计聚合接口，并扩展现有账单列表筛选参数。

后续可继续扩展：

- 账单发生时间从 `DATE` 升级为 `DATETIME`
- 分类增加颜色字段
- 将数据库字段从 `DECIMAL(10,2)` 进一步收敛为整数类型，和当前“分”的业务口径完全一致
