# 月账单统计前端接口对接文档

## 1. 对接提示

请前端 agent 先阅读产品需求文档，再按本文档对接后端接口。

重点提示：

- 所有接口路径都有全局前缀 `/api`。
- 统计接口和账单接口都需要 JWT 登录态。
- 后端响应会被统一包装为 `{ code, data, message, path }`。
- 前端真实业务数据在 `response.data.data` 中。
- 金额单位是“分”，不是“元”。
- 前端展示金额时需要 `amountCents / 100` 后格式化为 `¥x.xx`。
- 创建/更新账单时统一传 `amountCents`；不要再传 `amount`。
- 后端返回的支出、收入汇总金额都是正数。
- 单笔账单展示时，前端根据 `type` 决定显示 `-` 或 `+`。
- 当前账单发生时间字段是 `billDate`，精度到日期，没有时分秒。
- 当前分类模型没有颜色字段，前端不能依赖 `categoryColor`。
- 当前后端没有单独的账单标题字段，最近账单里的 `title` 暂时等于分类名称。
- “其他”分类的 `categoryId` 为字符串 `other`，用于兼容缺失分类。

## 2. 通用响应格式

成功响应：

```json
{
  "code": 0,
  "data": {},
  "message": "success",
  "path": "/api/statistics/monthly?year=2026&month=6&type=expense"
}
```

失败响应：

```json
{
  "code": 400,
  "message": "不能查询未来月份",
  "error": "BAD_REQUEST",
  "path": "/api/statistics/monthly?year=2027&month=1&type=expense"
}
```

## 3. 月账单统计接口

### 3.1 接口

```http
GET /api/statistics/monthly
```

### 3.2 查询参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| year | number | 是 | 年份，例如 `2026` |
| month | number | 是 | 月份，1 到 12 |
| type | string | 是 | `expense` 或 `income` |

### 3.3 示例请求

```http
GET /api/statistics/monthly?year=2026&month=6&type=expense
```

### 3.4 data 返回结构

```ts
interface MonthlyStatisticsData {
  year: number
  month: number
  type: 'expense' | 'income'

  summary: {
    /** 当前收支类型总金额，单位：分 */
    currentTypeAmountCents: number
    currentTypeCount: number
    /** 本月支出总金额，单位：分 */
    expenseAmountCents: number
    expenseCount: number
    /** 本月收入总金额，单位：分 */
    incomeAmountCents: number
    incomeCount: number
  }

  trend: Array<{
    year: number
    month: number
    /** 当月当前收支类型总金额，单位：分 */
    amountCents: number
    count: number
  }>

  categories: Array<{
    categoryId: string
    categoryName: string
    categoryIcon: string
    /** 当前分类总金额，单位：分 */
    amountCents: number
    count: number
    /** 当前分类占当前类型总金额的百分比，例如 39.35 表示 39.35% */
    percentage: number
  }>

  recentBills: Array<{
    id: string
    type: 'expense' | 'income'
    /** 单笔账单金额，单位：分 */
    amountCents: number
    categoryId: string
    categoryName: string
    categoryIcon: string
    title: string
    remark: string
    occurredAt: string
    billDate: string
    createdAt: string
  }>
}
```

## 4. 账单列表接口

分类账单明细页和本月全部账单页复用现有账单列表接口。

### 4.1 接口

```http
GET /api/bills
```

### 4.2 查询参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| month | string | 否 | 月份，格式 `YYYY-MM`；也兼容 `YYYY-MM-DD`，后端会截取前 7 位 |
| type | string | 否 | `expense` 或 `income` |
| categoryId | string | 否 | 分类 ID |
| pageNo | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页数量，默认 20，最大 100 |

### 4.3 本月全部账单请求

```http
GET /api/bills?month=2026-06&type=expense&pageNo=1&pageSize=20
```

### 4.4 分类账单明细请求

```http
GET /api/bills?month=2026-06&type=expense&categoryId=1&pageNo=1&pageSize=20
```

注意：

- 如果要查询“其他”分类，目前不要直接调用 `categoryId=other`。
- `other` 是统计接口聚合出的虚拟分类，现有账单列表接口只支持真实分类 ID。
- 如果前端第一版需要展示“其他”分类明细，需要后端再补充专门支持。

### 4.5 data 返回结构

```ts
interface BillListData {
  bills: Array<{
    id: string
    categoryId: string
    /** 单笔账单金额，单位：分 */
    amountCents: number
    type: 'expense' | 'income'
    remark: string
    billDate: string
    categoryName?: string
    categoryIcon?: string
    createdAt: string
    updatedAt: string
  }>
  total: number
  pageNo: number
  pageSize: number
}
```

## 5. 前端展示建议

### 5.0 账单写入金额

创建或更新账单时推荐使用 `amountCents`：

```json
{
  "categoryId": "1",
  "amountCents": 2800,
  "type": "expense",
  "remark": "午饭",
  "billDate": "2026-06-13"
}
```

注意：`amountCents` 是“分”，不是“元”。不要向账单写入接口传 `amount`。

### 5.1 金额格式化

接口金额单位是分：

```ts
function formatMoney(amountCents: number) {
  return `¥${(amountCents / 100).toLocaleString('zh-CN', {
    minimumFractionDigits: 2,
    maximumFractionDigits: 2,
  })}`
}
```

单笔金额符号：

```ts
function formatBillMoney(amountCents: number, type: 'expense' | 'income') {
  const sign = type === 'expense' ? '-' : '+'
  return `${sign}${formatMoney(amountCents)}`
}
```

### 5.2 月份切换

- 默认请求当前自然月、`type=expense`。
- 切换月份时保留当前 `type`。
- 不能让用户选择未来月份。
- 点击趋势柱状图时，使用柱体对应的 `year`、`month` 重新请求统计接口。

### 5.3 收支切换

- 切到支出：请求 `type=expense`。
- 切到收入：请求 `type=income`。
- 后端每次只返回当前类型的趋势、分类和最近账单，但 `summary` 同时包含收入和支出总览。

### 5.4 空状态判断

- 当前类型无数据：`summary.currentTypeCount === 0`。
- 无分类统计：`categories.length === 0`。
- 无最近账单：`recentBills.length === 0`。
- 趋势图始终渲染 `trend` 的 6 项，即使金额全为 0。

### 5.5 错误处理

- 请求未来月份会返回 400。
- 接口失败时展示“月账单加载失败，请稍后重试”。
- 不要无限自动重试。

## 6. 前端 agent 任务提示词

可以把下面这段连同需求文档一起给前端 agent：

```text
请基于《月账单统计详细需求文档.md》和《月账单统计前端接口对接文档.md》实现小程序月账单统计前端。

特别注意：
1. 后端所有金额字段都用 amountCents 命名，单位都是“分”，展示时必须除以 100。
2. 统计接口返回被全局包装，业务数据在 response.data.data。
3. 月账单首页使用 GET /api/statistics/monthly。
4. 本月全部账单页和分类账单明细页优先复用 GET /api/bills。
5. 统计接口里的 categoryId=other 是虚拟分类，当前账单列表接口暂不支持直接查询 other 明细。
6. 当前账单只有 billDate 日期字段，没有时分秒。
7. 当前分类没有颜色字段，不要依赖 categoryColor。
8. 不要实现年度账单、同比环比、预算、分享导出等非本期需求。
9. 快速切换月份或收支类型时，以最后一次请求结果为准，避免旧请求覆盖新数据。
```
